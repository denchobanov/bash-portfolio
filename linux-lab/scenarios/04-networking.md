# Scenario 4: Networking Troubleshooting

## Objective
Diagnose and resolve basic network connectivity issues between two virtual machines.

## Steps performed
- Checked IP address of the server using `ip a`
- Verified network connectivity using `ping`
- Tested SSH connection between client and server
- Identified and confirmed correct subnet configuration
- Resolved connectivity issues by ensuring both VMs are on the same network

## Commands used
- ip a
- ping <server_ip>
- ssh username@server_ip
- ip route

## Result
Network connectivity between the two virtual machines was successfully tested and confirmed. SSH access was validated after verifying correct IP configuration and network setup.
